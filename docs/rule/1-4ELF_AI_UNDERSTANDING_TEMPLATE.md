# 1-4ELF_AI_UNDERSTANDING_TEMPLATE - 專案理解與自評模板

> 本文件提供「可以直接複製貼上使用」的空白模板，搭配下列規則檔一起使用：
>
> - `1-2ELF_AI_ONBOARDING`：AI 上手分階段規則
> - `1-3ELF_AI_0-100_LEVELS`：四階段行為與輸出總表
>
> 本檔不新增任何規則，只提供格式與欄位，方便人類與 AI 在每個專案中快速建立紀錄。

---

## 1. 0–50 階段「掃描紀錄」模板

> 用於記錄 AI 在 0–50 分探索期實際讀了哪些檔案、初步印象與後續計畫。若使用工具產出完整專案目錄樹，請只擷取 1–2 層摘要與關鍵節點貼入下方模板，不要直接貼入包含資料庫檔案、build 成品等在內的完整樹。

```markdown
### 掃描紀錄 YYYY-MM-DD 第 1 輪

- 專案 / Repo 名稱：
- 本輪掃描時間：

- 專案目錄樹（建議使用 Tree Maker 產生後，人工擷取 1–2 層摘要；在檔名後用 [ ] / [x]
  表示是否已閱讀）：

    . 📂 <PROJECT_ROOT> ├── 📄 LICENSE [ ] ├── 📄 README.md [ ] ├── 📄 README.zh-CN.md [ ] ├── 📂
    assets/ [ ] ├── 📂 docker/ [ ] ├── 📂 docs/ [ ] ├── 📂 extensions/ [ ] ├── 📂 packages/ [ ] ├──
    📂 public/ [ ] ├── 📂 scripts/ [ ] ├── 📂 storage/ [ ] └── 📂 templates/ [ ]

- 初步印象（允許推測，請明確標註「推測」）：
    - 專案大致用途：
    - 主要技術棧初步判斷：

- 尚未閱讀的區塊（重要但還沒看；同時作為後續閱讀清單）：
    - [ ] 範例：`packages/api/**`
    - [ ] 範例：`packages/web/**`

- 下一輪閱讀計畫：
    - 下一步優先閱讀哪些檔案或資料夾：
    - 預計想回答的問題（例如：API 如何分模組？前端路由怎麼切？）：

- （可選，通常在進入 50 分以後使用）功能 / Flow 理解狀態（功能層）：
    - [ ] 範例：`UI-FLOW-LOGIN`
    - [ ] 範例：`UI-FLOW-AI-CHAT`
```

## 2. 專案理解進度勾選表模板

> 建議複製到該專案的 `PROJECT_OVERVIEW.md` 或單獨一個 `UNDERSTANDING_PROGRESS.md`
> 檔案中，讓人類與 AI 都能一眼看出目前理解階段。

```markdown
## 專案理解進度（AI 自評）

> 說明：這不是 TODO，而是「AI 對專案理解程度」的快照。

### 階段一（0–50）：探索期

- [ ] 已完成初步掃描（README、主要目錄結構）
- [ ] 已寫一份「掃描紀錄 / 初步印象」note
- [ ] 已建立一份「後續要閱讀檔案清單」，以 `- [ ] 檔案路徑` 方式列出，閱讀完成後逐項打勾

### 階段二（50–70）：模組成形

- [ ] PROJECT_OVERVIEW 初版已產出
- [ ] FRONTEND_ARCH 初版已產出（若有前端）
- [ ] API_OVERVIEW 初版已產出（若有需要）
- [ ] 至少 1–3 個關鍵功能的 UI-FLOW-<FEATURE>.md 已完成

### 階段三（70–90）：整體成形

- [ ] TECH_STACK_OVERVIEW 已建立，列出前後端/DB/Infra/DevOps 技術
- [ ] ARCH_OVERVIEW 已建立，包含整體架構 Mermaid 圖
- [ ] （選用）DEPLOYMENT_OVERVIEW 已建立或已在 PROJECT_OVERVIEW 中說明
- [ ] 早期零散筆記的有效內容已搬進上述總覽檔

### 階段四（90–100）：成熟維運

- [ ] 定期檢查並更新所有總覽文件（Overview / Arch / Tech Stack / Deployment）
- [ ] 每次重大變更（架構/技術棧）都有回寫到對應文件
- [ ] 已能為新成員 / 新 AI 指出「先讀哪幾份文件」作為入口
```

## 3. AI 自評與人類評分模板

> 搭配 `1-2ELF_AI_ONBOARDING` 第 6 節使用，讓 AI 與人類都能明確標示目前處於哪一階段。

```markdown
## AI 自評（理解階段）

- 日期：
- 自評階段（擇一勾選）：
    - [ ] 階段一（0–50）：探索期
    - [ ] 階段二（50–70）：模組成形
    - [ ] 階段三（70–90）：整體成形
    - [ ] 階段四（90–100）：成熟維運

- 本輪主要閱讀 / 操作內容（簡要列出讀了哪些檔、做了哪些動作）：

- 本輪回答中「最可能錯的 1–2 個假設」：
    -   1.
    -   2.

- 對部署 / Infra 理解的評估階段：\_\_/4

- 其他評語與建議：
    - ...
```
