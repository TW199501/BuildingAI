### [分析與方案]

- 分析：
    - 前端的登入行為已大致歸納為三條子流程：
        - 帳號登入（AccountLogin：帳號／密碼／註冊／忘記密碼）。
        - 微信掃碼登入（透過 `apiGetWxCode` + `apiCheckTicket` 輪詢 ticket）。
        - 強制手機綁定（`login-bind.vue` + `apiSmsSend` + `apiUserMobileBind`）。
    - 這些流程最終都會導向 `userStore.login(token)` 或 `tempLogin(token)`，因此關鍵在於：
        - token 在後端是如何產生與儲存的。
        - 失敗時以什麼錯誤碼／訊息向前端呈現，以便日後在前端做更好的錯誤 UX。
- 為避免一次讀太廣，本 todo 僅鎖定：
    - 與「登入」與「取得當前使用者資訊」直接相關的 API：
        - 帳號登入。
        - 微信登入 ticket 檢查與 code 取得。
        - 手機綁定（作為登入前置）。
        - `GET current user` 類 API。
    - 不延伸到一般 user CRUD、角色管理等 console 功能。

- 方案：依前端呼叫點分三大塊閱讀
    1. **帳號登入與取得當前使用者資訊**
        - 從 `@buildingai/service/webapi/user` 裡的登入與 `apiGetCurrentUserInfo` 入手：
            - 確認：
                - 登入 API 的 URL / method / request body。
                - 回傳的 `LoginResponse` 結構（包含 `token`、`hasBind`、`mobile` 等欄位）。
            - 在 `packages/api` 裡搜尋對應 controller／service：
                - 確認帳號登入時的：
                    - 使用者查詢與密碼驗證邏輯。
                    - 帳號不存在、密碼錯誤等錯誤碼與訊息。
                - 確認 `apiGetCurrentUserInfo` 回傳的使用者資訊結構（與前端 `UserInfo` 對應）。

    2. **微信掃碼登入 ticket 流程**
        - 從 `apiGetWxCode` 與 `apiCheckTicket` 入手：
            - 確認：
                - 取得 QR code 的 URL / method 與 response 結構（`url`、`key`）。
                - ticket 檢查 API 的 URL / method 與 response 結構（是否攜帶 `LoginResponse`）。
            - 在後端 auth/user 模組中找到對應實作：
                - 確認 ticket 產生與驗證邏輯。
                - 確認可能的錯誤狀態（ticket 過期、無效等）。

    3. **手機綁定與 SMS 發送流程**
        - 從 `apiSmsSend`、`apiUserMobileBind` 入手：
            - 確認：
                - SMS 送出 API（`scene: BIND_MOBILE`）的路徑、速率限制與錯誤行為（若有）。
                - 手機綁定 API 的 request/response 結構（是否只回成功／失敗）。
            - 在後端對應的 controller／service 裡確認：
                - 驗證碼產生、儲存與比對邏輯。
                - 常見錯誤（例如驗證碼錯誤／過期）。

- 完成閱讀後：
    - 在 `docs/ui-flow-login.md` 中新增「後端資料流摘要」小節：
        - 將上述三塊的主要 endpoint / DTO / 回應欄位以文字方式列出.
        - 保持細欄位仍由後端 API 文件維護，不在此處重複維護全量 spec.

### [實作重點摘要]

- 仍未修改任何程式碼，本 todo 僅進行閱讀與文件更新.
- 已完成的工作：
    - 明確定義本 todo 的範圍僅限 **Feature B（login/auth）相關的後端登入與使用者資訊 API 流程**.
    - 依原先規劃的三塊路線完成第一輪閱讀：
        - **帳號登入與取得當前使用者資訊**：
            - 前端 Web API：
                - `apiAuthLogin(params)` → `POST /auth/login`，回傳 `LoginResponse`（包含 `token` 與
                  `user`）。
                - `apiGetCurrentUserInfo()` → `GET /user/info`，回傳 `UserInfo`.
            - 後端：
                - `AuthWebController.login`（`@Public() @Post("login")`）呼叫
                  `AuthService.login`，驗證帳號與密碼後建立 token 與過期時間，並回傳包含角色與權限資訊的使用者物件.
                - `UserWebController.getUserInfo`（`@Get("info")`）透過 `userService.findOneById`
                  讀取使用者與角色資料，並以 `rolePermissionService.getUserPermissions`
                  算出是否擁有權限，回傳 `permissions` 標誌欄位.
        - **微信掃碼登入 ticket 流程**：
            - 前端 Web API：
                - `apiGetWxCode()` → `GET /auth/wechat-qrcode`，取得 `WechatLoginCode`（`key`,
                  `qrcode`, `url`）。
                - `apiCheckTicket({ key })` → `GET /auth/wechat-qrcode-status/:scene_str`，回傳
                  `WechatLoginTicket`（`is_scan` + `user: LoginResponse`）。
            - 後端：
                - `AuthWebController.getWechatQrcode`（`@Get("wechat-qrcode")`）呼叫
                  `WechatOaService.getQrCode` 產生帶場景值的臨時二維碼.
                - `AuthWebController.getWechatQrcodeStatus`（`@Get("wechat-qrcode-status/:scene_str")`）呼叫
                  `WechatOaService.getQrCodeStatus` 根據場景值回傳掃碼狀態與登入結果.
                - 另有 `wechat-callback`
                  相關 endpoint 由微信伺服器呼叫，更新 Redis 中的場景狀態，最終由 `getQrCodeStatus`
                  供前端輪詢使用.
        - **手機綁定與 SMS 發送流程（概略）**：
            - 前端 Web API：
                - `apiSmsSend({ scene, mobile })` →
                  `POST /sms/sendCode`，用於發送綁定手機的簡訊驗證碼.
                - `apiUserMobileBind({ type, mobile, code })` →
                  `POST /user/bindMobile`，在驗證碼正確時完成手機綁定，並由前端透過 `loginHandle`
                  再次進行正式登入流程.
            - 後端：
                - 本 todo 僅確認路徑與用途，尚未展開具體 controller/service 細節，保留未來若需強化安全／風控時再新開 todo 深挖.
    - 在 `docs/ui-flow-login.md`
      中新增「3.1 後端資料流與 API 摘要」小節，總結上述帳號登入／微信登入／手機綁定／使用者資訊 API 與 user
      store 狀態的關係.
