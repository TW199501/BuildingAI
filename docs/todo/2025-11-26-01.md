# TODO 2025-11-26 01

## 主題：多國語言擴充（繁體中文）與建置流程驗證

### [問題標題]

- 將 `zh-TW` 正式納入語系配置與翻譯流程，並驗證 UI / script 均能載入繁體翻譯。
- 整理今日修改（i18n、tsconfig、Docker compose）的步驟、影響與驗證結果，提供後續 build / 部署的參考。

### [背景與重現步驟]

- 既有系統僅有 `zh`、`en`、`jp`，UI 開啟仍預設顯示簡體。
- `pnpm run i18n:translate` 腳本採硬編碼語系，無法偵測新加入的 `zh-TW`。
- `extensions/buildingai-simple-blog` 的 `tsconfig.*.json` 缺少 `target`/`module`/`incremental` 等設定，導致 IDE typecheck 錯誤。
- Docker 預設從騰訊鏡像拉取 redis/postgres/nodejs，跨區下載較慢，需要能覆寫。
- 今日流程：完成 i18n 調整 → 修 tsconfig → 跑 lint/typecheck/build → 驗證 Docker up/down → 調整 compose → 補本文件。

### [相關檔案路徑與定位]

- i18n 設定：`packages/web/@buildingai/i18n-config/src/language.ts`、`@buildingai/i18n-config/src/i18n/index.ts`、`scripts/translate-i18n.mjs`
- Nuxt 設定：`packages/web/@buildingai/nuxt/src/nuxt.config.ts`
- App 翻譯：`packages/web/buildingai-ui/app/i18n/zh-TW/*`
- Extension tsconfig：`extensions/buildingai-simple-blog/tsconfig.api.json`、`tsconfig.web.json`
- Docker：`docker-compose.yml`
- Task tree：`docs/todo/2025-11-26-01.json`

### [分析與方案]

1. **語系與翻譯**
   - 依照 Task Tree 的語系盤點結果，`Language` enum、`languageOptions`、`sharedMessages` 已能掛上 `zh-TW`，但翻譯腳本與 Nuxt defaultLocale 尚未更新。
   - 方案：
     - `translate-i18n.mjs` 先以最小成本硬加 `zh-TW` 定義，使腳本立即生效；後續可再改寫為動態讀取 `language.ts`。
     - Nuxt `defaultLocale` 改為 `zh-TW`，並透過 cookie/plugin 初始化語言，確保第一次載入就顯示繁體。
2. **Extension tsconfig**
   - IDE 報錯 `--target`/`--module`/`tsBuildInfo`，根因是 `tsconfig.api.json` 及 `tsconfig.web.json` 欠缺完整 compilerOptions。
   - 方案：加入 `target`, `module`, `incremental`, `tsBuildInfoFile`，並改為 extend 正確的 Nuxt tsconfig。
3. **Docker compose**
   - 過去固定指向騰訊 registry。為方便在其他區域或有內網限制時切換鏡像，改用環境變數覆寫。
   - 方案：`docker-compose.yml` 的 `redis/postgres/nodejs` image 欄位改寫成 `${*_IMAGE:-預設}`。

### [實作重點摘要]

- **語系**
  - `language.ts`：已含 `Language.ZH_TW`、`languageOptions` 追加繁體，`navigatorMap` 補上 `"zh-tw"` 映射。
  - `@buildingai/i18n-config/src/i18n/index.ts`：加入 `zh-TW` shared messages imports。
  - `scripts/translate-i18n.mjs`：新增 `ZH_TW` 及 `"繁體中文"` option，translation code 使用 `zh-Hant`，`TARGET_LANGS` 現已包含 `zh-TW`。
  - `@buildingai/nuxt/src/nuxt.config.ts`：`defaultLocale` 改為 `zh-TW`，`detectBrowserLanguage` cookie 仍沿用 `nuxt-ui-language`。
  - App 端語言選單／cookie 流程：User Menu / Install / Profile 已能列出繁體並正確切換。
- **Extension tsconfig**
  - `tsconfig.api.json`：補 `target: es2020`、`module: esnext`、`incremental: true`、`tsBuildInfoFile: build/api.tsbuildinfo`。
  - `tsconfig.web.json`：改 extend `../../packages/web/buildingai-ui/.nuxt/tsconfig.json`，並記錄 `include`/`exclude`；`tsBuildInfoFile` 更名為 `build/web.tsbuildinfo`。
- **Docker compose**
  - `redis/postgres/nodejs` image 欄位改為 `${REDIS_IMAGE:-...}` 等，可透過 `.env` 或臨時環境變數覆寫來源，例如 `set POSTGRES_IMAGE=postgres:17.6`。
- **指令執行**
  - `pnpm run lint:fix`（根目錄）→ 36 個 package 皆成功。
  - `pnpm run typecheck` → `turbo` 全域 `check-types` 成功。
  - `pnpm build` → 36 個 package build 完成。
  - `pnpm docker:up` / `pnpm docker:down` 驗證容器可正常啟停；`docker logs buildingai-nodejs` 顯示首次啟動需等待 `pnpm install`。

### [驗證與結果]

- UI：重新啟動後首次開啟即為繁體，並可透過語言選單自由切換。`nuxt-ui-language` cookie 正常寫入 `zh-TW`。
- 翻譯腳本：於 `extensions/buildingai-simple-blog` 目錄執行 `pnpm run i18n:translate`，輸出 log 顯示 `zh-TW` 已列入目標語言。
- TypeScript：IDE 不再報 `--module`/`--target`/`tsBuildInfo` 錯誤，`pnpm run typecheck` 成功。
- Build & Docker：`pnpm build`、`pnpm docker:up` 皆於今日重新驗證；容器 `redis/postgres` 為 `healthy`，`nodejs` 在依賴安裝完成後能正常提供 4090 服務。
