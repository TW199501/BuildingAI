    - **後端 controller / DTO 層**：
      - `SystemConsoleController`（`packages/api/src/modules/system/controllers/console/system.controller.ts`）：
        - `@Public() @Post("initialize")` → `systemService.initialize(dto, ipAddress, userAgent)`.
      - `initializeDto`（`packages/api/src/modules/system/dto/system.dto.ts`）：
        - extends `RegisterDto`（去除 `terminal`），並新增 `avatar?`、`websiteName?`、`websiteDescription?`、`websiteLogo?`、`websiteIcon?`，全部附帶字串驗證註解.
    - **後端 service 層**：
      - `SystemService.initialize(dto, ipAddress?, userAgent?)`：
        - 使用 `userService.hashPassword` 加密密碼，透過 `userService.create` 建立 **超級管理員帳號**（`isRoot=YES`、`source=CONSOLE`、隨機或指定 avatar 等）。
        - 透過 `dictService.set` 寫入網站名稱／描述／Logo／Icon 至 `group: "webinfo"`.
        - 透過 `dictService.set("isInitialized", true, { group: SYSTEM_CONFIG })` 標記系統已初始化.
        - 使用 `rolePermissionService.getUserRoles` / `getUserPermissions` 取得角色與權限資訊.
        - 使用 `authService.userTokenService.createToken` 產出登入 token 與過期時間（帶入 `UserTerminal.PC`、IP 與 user-agent）。
        - 更新使用者 `lastLoginAt`.
        - 回傳物件：`{ isInitialized: true, token, expiresAt, user: { ...userWithoutPassword, role, permissions } }`.
        - 錯誤時使用 `HttpErrorFactory.badRequest(error.message, error.data)` 拋出，前端目前僅顯示 `message`.
  - 在 `docs/ui-flow-install.md` 中補充：
    - 初始化 API 的 request/response 主要欄位與端點（`POST /consoleapi/system/initialize`）。
    - 安裝過程中建立的資料（超級管理員帳號）與寫入的設定（`webinfo` 字典、`SYSTEM_CONFIG.isInitialized`）。
